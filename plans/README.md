## Findings considered and rejected

- **CORRECTNESS-01 (Azure callback timer leak)**: The timer is bounded by the
  `SIGN_IN_TIMEOUT_MS` (5 minutes) and the window for the leak requires an
  exception at a very specific synchronous point after `server.listen`. Impact
  is negligible in practice; not worth a plan.
- **CORRECTNESS-02 (headSha validation)**: Both GitHub and Azure DevOps APIs
  return a well-formed SHA as a required field; an empty headSha in production
  is near-impossible. Added a note in CONTEXT.md `RepoCache` entry instead.
- **DIRECTION-01 (Phase 11 — blog posts)**: Pure writing/marketing work with
  no code component. No implementation plan appropriate; maintainer executes
  directly from the ROADMAP checklist.
- **DIRECTION-03 (OAuth app IDs placeholder)**: Explicitly a user action
  (developer account + app registration), not a code change. The TODO in source
  is the correct artifact. Documented in `docs/setup-oauth-apps.md`.
- **TEST-04 (full-flow integration tests)**: Superseded by plans 004 (IPC
  handler tests) and 010 (golden AI tests), which together cover the critical
  integration seams without a separate integration test plan.
- **TEST-05 (golden tests)**: Covered by plan 010.
- **DX-03 (CLAUDE.md missing)**: `CLAUDE.md` exists at the project root at
  commit `1278cf3` — the finding was based on a stale ARCHITECTURE.md
  reference. Not a real gap.
- **PERF findings (WorkspaceScreen DiffRow memoization)**: The DiffRow
  component renders inline styles without `React.memo`. At realistic PR sizes
  (< 2,000 changed lines) this is unnoticeable. Plan 007's split is the
  prerequisite; memoization can be added per-component then if profiling shows
  a need. Not worth a standalone plan now.
